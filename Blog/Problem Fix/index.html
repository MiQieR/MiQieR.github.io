记一次电脑故障的解决方法

发表于2021-09-16|更新于2022-05-12|Blog

Lecture 1: LANs Technology -Switched LANs

1.1

Shared media LANs使用Hub，half-duplex mode；Switched LANs使用Switch，full-duplex mode

LAN Switch：Layer 2，包括Application-specific integrated circuits (ASICs)，Small form-factor pluggable (SFP)，Power of Ethernet (PoE)

Layer 2 LAN switch：仅基于第 2 层 MAC 地址执行交换和过滤；Layer 3 LAN switch：可以了解与其每个端口关联的 MAC 地址和 IP 地址

1.2

Why redundancy冗余? To eliminate network outages caused by a single point of failure.

但可能产生layer 2 loop (switching loop)

解决方法：允许物理环路，但不允许loop logical topology，即tree

The spanning-tree protocol (STP) 利用专用算法来发现交换网络中的物理环路并实现loop free logical topology

Spanning-tree algorithm：

非根节点Non-root Bridge只能有一条路径通往 根节点Root bridge，不属于树的冗余链接将被阻止

生成树算法允许一些端口处于转发状态，而将一些其他端口置于阻塞状态。该算法确定每个交换机中端口的角色和状态：

▪ The Root ports and the Designated ports are used for forwarding (F) the traffic.根端口和指定端口用于转发 (F) 流量。
▪ The Non-designated ports are used for blocking (B) the traffic.非指定端口用于阻塞 (B) 流量。

当交换网络的树形拓扑结构稳定时，存在以下元素：

每个网络一个Root bridge；每个Non-root bridge一个Root port；每segment段一个Designated port指定端口；Unused ports are Non-designated ports未使用的端口是非指定端口

Bridge Protocol Data Unit (BPDU)

STP 要求所有交换机交换消息以检测交换环路并创建无环路逻辑拓扑。这些消息称为桥接协议数据单元BPDU

默认情况下，每两秒发送一次 BPDU

每个BPDU包含：Root BID，Root Path Cost，Sender BID，Port ID

BPDUs 继续在被阻塞的端口上接收，这确保了如果活动路径或设备发生故障，可以重新计算生成树算法

Bridge ID (BID)

BID：8 byte= 2 byte（十进制数）Bridge Priority 优先级 + 6-byte (十六进制) MAC address（0～65,535；default 32,768）

Extended system ID

当STP支持VLAN时，使用Extended system ID：

2byte(= 4bits Bridge Priority 优先级 + 12bits Extended system ID) + 6byte MAC = Bridge ID with Extended system ID

网桥优先级值只能是 4096 的倍数

Port ID

用于确定到Root bridge的最佳路径

2 byte = 8bits Port Priority + 8bits Port Number

Lower Port IDs are preferred over higher Port IDs in the STP decision process. Port ID低的比高的优先级高

configurable可配置，0～255，默认128

Path Cost

STP 使用Path Cost来确定到根网桥的最佳路径。路径成本是两个网桥之间路径中链路成本的总和。最低的累积路径成本将是通往根网桥的最佳路径。（整数存储）

Spanning-tree protocol operations

STP 操作四个步骤：1.选择根网桥。2.在每个非根网桥上选择一个根端口。3.在每个网段上选择一个指定端口。4.为所有未使用的端口选择非指定端口。
最终，STP 在交换网络中构建树形拓扑。

Step 1: Elect a Root bridge

拥有最低BID的为Root bridge

Step 2: Elect the Root ports

Non-root bridge选一个到Root bridge路径成本最低的端口，如果多个端口具有相同的最低路径成本，则接收到具有最低 Sender Bridge ID，若Sender Bridge ID仍一样，则选最低的Sender Port ID的端口成为 Root 端口

小陷阱：不要直接对比BID哪个最小，要先看连在先发的谁上

Step 3: Elect the Designated ports

每个网段上选择一个到根网桥的路径成本最低的指定端口，如果多个端口具有相同的最低路径成本，则BID最低的端口将成为指定端口。

同一Switch上，Root端口连向Root bridge，Designated端口连向树的子节点Switch

Step 4: Elect the Non-designated ports

STP 选择所有未使用的端口作为非指定端口。一个Designated 上级端口对应一个下级Non-designated port，所以一条逻辑上被截断的线只有一个port blocked by STP

STP 停止该链路的任一端口被阻塞的链路，从而构建tree topology

当设备或链路出现故障时，STP 会重新计算拓扑

STP states 5个

Disabled：被管理员禁用，No BPDUs heard, No frames forwarded

Blocking：被STP阻断，BPDUs heard, No frames forwarded

Listening：搭建topology，BPDUs heard, No frames forwarded

Learning：搭建switching table，BPDUs heard, No frames forwarded

Forwarding：发送和接收数据，BPDUs heard, Frames forwarded

顺序：Blocking -> Listening -> Learning -> Forwarding

Lecture 2: LANs Technology - Virtual LANs

2.1 Virtual LANs

用户可以逻辑上管理LAN而不用从物理上管理

同一个VLAN = 同一个IP subnet <-> 同一个boardcast domin

switch上每个物理port可分给一个或多个VLAN

Switched ports

陷阱：switch port协议最高layer2；layer3 port是route port

switch port分为access port和trunk port

access port trunk port
区别 只给某一个VLAN 指派多个VLAN
一般用于 Switch-PC间的port Switch-Switch间的port
协议 IEEE 802.3 or called Ethernet IEEE 802.1Q or Cisco ISL

trunk port的协议：是非标准的layer2 protocol：trunking protocol [IEEE 802.1Q 或 Cisco ISL] -> 为原数据包加包头Tag[记录VLAN ID]，发送时加Tag，接受时去除Tag

VLAN communications

Intra - VLAN：同一VLAN内，第二层；Inter-VLAN：VLAN之间，第三层

连接两个VLAN：router(gateway) 或 layer 3 switch（通常用后者）

​ ⬆️若router连2个VLAN的物理port：必须trunk

​ ⬆️陷阱：只有switch的port成为trunk port；route是第三层，不可称为trunk port

2.2 Redundant virtual LANs

Link redundancy链路冗余/链路聚合：多个物理link连成一个逻辑link（作用：提供更高的吞吐量，并且在其中一条物理链路发生故障时提供冗余，甚至在两台交换机之间的冗余路径上提供负载平衡）【例子：EtherChannel：所有端口在两端交换机的所有接口上必须具有相同的速度、双工模式和 VLAN 信息，EtherChannel协议：LACP和PAgP】

Gateway redundancy：一个VLAN仅有一个gateway是active，其余均为它的stand by【不使用物理gateway IP和MAC，而使用virtual IP和MAC（比如实验中的.254)】

Lecture 3: LANs Technology - Wireless LANs

Wireless LANs(WLANs)使用radio frequencies (RF)传输，通过access point (AP)连接

仅在layer1 2与有线有区别，仅关注IEEE 802.11标准

3.1 Wireless LAN standards

2.4 GHz channel scheme

美国到11channel，欧洲和亚洲到13channel，日本到14channel

2.4G每个channel间隔5Mhz，占用22Mhz

non-overlapping channel：（完全不相交的Channel）2.4G有最多3个，5G有8个

Spread Spectrum

扩频技术Spread Spectrum：channel内将物理带宽划分为逻辑带宽：DSSS和OFDM（前者古老且较慢，后者为现在常用，2Mhz一个）

Modulation

modulation调制：表示二进制

Spread Spectrum + modulation才可在channel内通信

解决冲突

CSMA/CA ->监听（明文）到发送的间隔有：DIFS PIFS SIFS（长 <-> 短），有线是定长的

无线特殊冲突：hidden node（两个AC不知道对方存在） 解决方法：RTS/CTS 先请求再发送

3.2 Wireless LAN operations

WLAN devices

包括 wireless adapter 和 access point 和 antenna

AC不同连线使其成为不同Role： AC连AC：Root ｜ AC连无线PC：Repeater中继 ｜ AC连有线PC：Bridge桥接

antenna

omni-directional antenna全向天线 ｜ directional antenna有向天线

Isotropic理想天线，球型 ｜ Dipole耦形天线

相同距离，仅水平方向天线能量密度SSd > 球形天线SSi，增益:理想天线<全向天线<有向天线

WLAN topologies （3种）

Ad hoc mode：多个无线客户端无需AP即可连接，如PC连PC

Infrastructure mode：所有无线客户端通过一个或多个AP 互连（包含两种）

其中，Ad hoc mode分为independent basic service set (IBSS) 或 ad hoc network

Infrastructure mode中：

basic service sets (BSS) 网络由一个连接所有相关无线客户端的单个 AP 组成（每个 AP 的第 2 层 MAC 地址，称为基本服务集标识符 (BSSID)，用于唯一标识每个 BSS）

extended service set (ESS) 网络是一个BSS的RF覆盖范围不足时，由多个AP组建的（每个 ESS 由一个 SSID（也称为 ESSID）标识，在 ESS 中，每个 BSS 由其 BSSID 标识）

WLAN process

Probing -> Authentication -> Association

Authentication阶段：

Open authentication：NULL authentication

Wired Equivalency Protection (WEP)：a pre-shared key authentication mechanism，带有静态密钥的 RC4 加密方法

IEEE 802.11i standard：for WLAN 认证和加密

认证：Extensible Authentication Protocol (EAP)

加密：WPA（使用TKIP）或 WPA2（使用AES）

    Helper IP = Server_IP

4.2 Address Translation

NAT address terminology

Inside local address: 内部网络上管理员分配给主机的 IP 地址

Inside global address: 代表外部网络的Inside local address的IP 地址

Outside local address: 内部网络上的主机已知的外部主机的IP 地址

Outside global address: 分配给外部网络主机的IP 地址

NAT types

Static NAT: local 和 global address一对一映射

Dynamic NAT: 将私有 IP 地址映射到公共 IP 地址池中的 IP 地址

Dynamic NAT Overloading/Port Address Translation (PAT): 将多个私有 IP 地址映射到一个公共 IP 地址，每个私有地址都由一个端口号跟踪

Lecture 5: WANs Technology - Dedicated Networks

5.1 WAN Technologies

WAN：physical layer and data link layer of the OSI reference model

WAN 系统包括：

Customer premises equipment (CPE)：物理上位于用户站点上的设备

Modem (DCE)：使终端用户设备适应本地环路的设备

WAN switch：(Provider’s Device)

用Serial串行线而不是Parallel并行线：因为很难做同步

DTE 到 DCE 信号中使用的八个引脚可以按其功能分为三类：

类别 包含的技术
Data transfer group TxD，RxD，SG
Hardware flow control RTS，CTS
Modem control DTR，CD，DSR

DTE 到 DTE之间建立连接：

High-level Data Link Control (HDLC) 或 Point-to-Point Protocol (PPP)

PPP

PPP由两个子协议组成：

Link Control Protocol：用于建立或终止 PPP 连接，并协商建立控制选项 （Data Link Layer 2）

Network Control Protocol：用于封装和协商多个网络层协议的选项（Network Layer 3）

PPP session establishment

  1. Link-establishment phase 使用LCP协议

  2. Authentication phase使用PAP两次握手明文发送 不安全 / CHAP三次握手不用明文 安全

  3. Network layer protocol phase 使用NCP协议

5.2 Dedicated Networks

WAN types

特点 技术
Dedicated network pre-established physical circuit；dedicated bandwidth ▪ T1/E1, T3/E3 lease line
▪ xDSL
▪ SONET
Circuit-switched network physical circuit on demand；dedicated bandwidth during the date transfer period ▪ POTS
▪ Narrowband ISDN
Packet-switched network no physical circuit；bandwidth not dedicated and is shared with other users ▪ X.25
▪ Frame Relay

允许多条logical circuit存在于同一个physcial circuit 即为virtual circuit(Packet-switched network)

Lecture 6: WANs Technology - Circuit-switched Networks

6.1 Circuit-switched Networks

Circuit-switched 与静态最大的不同在于允许用户自行决定谁与谁连接

早期网络中使用Layer1 signal，现在用Digital Signaling

包括：Continuous connections，Dial-up network connections，Optical circuit switching

6.2 Narrowband ISDN

窄带 ISDN 有两种基本类型 组成 带宽
Basic Rate Interface (BRI) 2x64Kbps Bearer (B) channels for voice and data + 1x16Kbps Delta (D) channel for control and signaling information 144Kbps
Primary Rate Interface (PRI) in North America and Japan (T1) 23x64Kbps B channels + 1x64Kbps D channel 1.544Mbps
Primary Rate Interface (PRI) in Europe, Australia, and other parts of the world (E1) 30x64Kbps B channels + 1x64Kbps D channel + 1x64Kbps Framing channel 2.048Mbps

ISDN BRI 设备包括：

Terminal Equipment 1 (TE1)：指定与 ISDN 网络兼容的设备，TE1 连接到Type 1/2 NT

Terminal Equipment 2 (TE2)：指定与 ISDN 不兼容且需要 TA 的设备

Terminal Adapter (TA)：将标准电信号转换为 ISDN 使用的形式，以便非 ISDN 设备可以连接到 ISDN 网络

NT Type 1 (NT1)：将四线 ISDN 用户线路连接到传统的两线本地环路设施

NT Type 2 (NT2)：引导流量进出不同的用户设备和 NT1， NT2是一种智能设备，可以进行切换和集中

ISDN protocols

The ISDN physical layer：

The ISDN data link layer ：（封装控制指令）

The ISDN network layer：（通过电话号码找对方）

D channel 第二层中有 LAPD协议 Q.921，第三层Q.931；B channel 第二层PPP，第三层IP。以D channel搭建好临时circuit，然后用B channel传输

Lecture 7: WANs Technology - Packet-switched Networks

7.1 Packet-switched Networks

Switching Method Pros Cons Key Features
Circuit-switched Offers a dedicated transmission channel that is reserved until it is disconnected. Dedicated channels can cause delays because a channel is unavailable until one side disconnects. Uses a dedicated physical link between the sending and receiving devices.
Packet-switched Packets can be routed around network congestion. Packet switching makes efficient use of network bandwidth. Packets can become lost while taking alternative routes to the destination. Messages are divided into packets that contain source and destination information. Virtual circuit uses a logical connection between the source and destination device

7.2 Frame Relay

如何命名和构建virtual circuit。命名：DLCI，每一条circuit的名字不同；构建：switching table内的In_DLCI和Out_Port…

Frame Relay topology

Full mesh：每个站点都连接到每个其他站点

Star topology (hub and spoke)：每个远程站点都有一个通过单个 VC 访问frame relay的链接。Hub有一个带有多个 VC的访问链路，每个远程站点一个

Partial mesh：比Star topology多，但少于Full mesh

Access rate / Port speed

Committed information rate (CIR)：服务提供商承诺提供的保证速率

Excess information rate (EIR)：guaranteed rate of service provider commits to providing

Committed Time (Tc)：计算费率的时间间隔

Committed burst (Bc)：The maximum number of committed bits that the switch agrees to transfer during a time interval Tc。交换机同意在时间间隔 Tc 内传输的最大承诺比特数

Excess burst (Be)：The maximum extra number of uncommitted bits that the Frame Relay switch attempts to transfer beyond the CIR。帧中继交换机尝试传输超出 CIR 的最大额外未提交比特数

如果计数器低于 Bc，则该帧不符合丢弃条件（DE=0）。

如果将计数器置于 Bc 之上，则该帧有资格被丢弃（DE=1），这表明如果网络拥塞或网络容量不足，则可能会丢弃帧。

如果计数器超过 Bc + Be，则丢弃discarded该帧。

Frame Relay flow control

FECN：当Frame Relay switch识别到网络中的拥塞时，它会向目标设备发送一个 FECN 数据包，表明发生了拥塞

BECN：当Frame Relay switch识别到网络中的拥塞时，它会向源路由器发送一个 BECN 数据包，指示路由器降低它发送数据包的速率。如果路由器在当前时间间隔内收到任何 BECN，它会将传输速率降低 25%。
