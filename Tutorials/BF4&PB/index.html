Lecture 8 Secure Tunnel

8.1 Secure Tunnel

Tunneling: 进入Tunnel时加上Outer Header，出Tunnel时去掉Outer Header （考试时：电脑A发给B，数据包从哪个网口出来时，是加还是去掉Header）

唯一Can protect you data的Tunneling协议：IPsec

IPsec的特性及使用什么算法保护（来自8.2 VPN Security）：

特性 算法 应用
Confidentiality机密性 cryptography(encryption) algorithm DES、3DES 或 AES
Integrity完整性 hash algorithm MD5 或 SHA
Authentication身份验证 authentication method PSK 或 RSA
Anti-replay防重放攻击 sequence number

IPsec的sub-protocols：Authentication Header (AH) - Protect data from writing；Encapsulated Security Payload (ESP) - Protect data from reading

它们都有两种模式：transport mode和tunnel mode [Tunnel Mode加2个Header，Transport Mode只加1个]

Transport mode在IP Header后加AH/ESP Header；Tunnel mode在IP Header前加AH/ESP Header，并在AH/ESP Header前加New IP Header；ESP还会加ESP Trailer和ESP Auth

8.2 Virtual Private Network

VPN的type：Site-to-site VPN 公司路由和公司路由间；Remote-access VPN 各个主机和公司路由之间 [两种用的都是Tunnel Mode，所以都有2个Header]

看原PPT，会考Outer Header和Inner Header里的地址分别是 _ 和 _ :

Site-to-site VPN：

PC A1 -| |-B1 PC

​ |——-R1~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~R2——-|

PC A2 -| |-B2 PC

如果A1->B1：Outer Header=R1->R2; Inner Header=A1->B1

Remote-access VPN：

PC A1 -| |-C1 PC

​ |——-R1~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~ ~|

PC A9 -| |-C2 PC

如果C1->A9：C1会有2个地址，Inner Header里是Private Address，Outer Header里是Original Address。所以Outer Header=C2[Original Address]->R1; Inner Header=C2[Private Address]->A9

参考Exercise 8.1

Create VPN的步骤：

Step 1: Interesting traffic initiates the IPsec process

Step 2: IKE phase 1 - generate the IKE SA for IKE phase 2

Step 3: IKE phase 2 - generate the IPsec SA for data transfer真正要传的数据

Step 4: Data transfer between IPsec peers

Step 5: IPsec tunnel termination

【IKE phase 1保护IKE phase 2，IKE phase 2保护要传的数据】

创建VPN隧道时通常用Diffie-Hellman Exchange (DH)创建IPsec密钥

一定不会交换Private key，只交换Public key。例：生成User2的Secret key用User2 Private key和User1 Public key

【计算题】P, g已知。User1 Private key=X₁<P; Public key=Y₁=g^x₁ | User2 Private key=X₂<P; Public key=Y₂=g^x₂ | Secret key = Y₂^x₁ mod P = Y₁^x₂ mod P

9.2 Access Control

Access Control 是 Authorization 的一种应用

Access Control最著名的应用是Firewall

区分两种Firewall

Application gateway firewall Packet filtering firewall
layer application layer（7） network layer 和 transport layer （2,3）
区别 在更深的应用层所以更好安全性 在较低层所以性能更好
连接数 1st & 2nd connections one connection

Firewall的3个area：internal，external，DMZ。（会考设备在Firewall的哪个area）

Access Control Lists (ACLs)：属于Packet filtering firewall

ACL判断数据能否通过，要考察的数据：source and destination IP address, source and destination port number, layer 3 protocols（source and destination addresses, protocols, and upper-layer port numbers)只能是2、3层的，超出这两层的数据看不到

ACL statements从上往下匹配，匹配到合适的就直接使用，忽略后面的

ACL只需inbound或者outbound一种即可

inbound: before the message come into the device to make decision permit or deny

outbound: what message can exit this interface

Wildmark masks

【计算题】注意指的一台电脑还是一个子网（hosts=subnet，servers=一台电脑），前者Wildmark masks=0.0.0.0，后者Wildmark masks = 子网掩码取反

例：Subnet1有台电脑192.168.2.222/26，得出子网IP：192.168.2.192，子网掩码255.255.255.192，Wildmark mask=0.0.0.63

Lecture 10 Traffic Control

10.1 Traffic Shaping

流量整形（“平滑”输出流量）用于控制接口上的输出流量以匹配远程目标接口的流量速度，从而消除由数据速率不匹配引起的拓扑瓶颈。这是通过在流的数据包之间引入额外的延迟来实现的，以更均匀地分隔数据包，从而减少流的“突发”。

Token bucket algorithm

两个参数Token generator with rate: r 和 Token bucket with size: b

Only have available token can send the message

大量关于流量的计算

10.1 Traffic Queuing

将拥塞的路由器配置为对数据包进行优先级排序，以便首先发送任务关键型和延迟敏感型流量

不同的Queuing Methods：（少量计算）

  1. First In, First Out (FIFO) 不包含优先级或流量类别的概念。缺点：Big traffic占据大部分时间，small traffic没时间发送 超时
  2. Weighted Fair Queuing (WFQ)
  3. Priority Queuing (PQ) 缺点：最低优先级的数据可能永远没机会发

【计算题】Order：

（首先将同一IP、端口号、协议的数据包放入同一Queue）

FIFO：按照进入Queue的顺序发出

WFQ：每个队列都有机会发。先按照Weight规定的顺序发第一个包①，并记录包①大小x权重，然后按照Weight顺序看下一级有没有大小x权重比它小的②③④，如果有，发②；如果发完②发现②大小x权重 < ①大小x权重，则发②同权重的下一个③，若发完③发现 ②大小x权重+③大小x权重 > ①大小x权重，则按照Weight顺序看下一级…

PQ：高优先级的先发，同一优先级的按FIFO发

Lec11 Network Monitoring

network management submodels:

  1. Organization: manager, agent [components of network management]
  2. Information: management information base (MIB), SMI决定MIB中的语法和语义 [structure and storage of network management information]
  3. Communication: transport protocol, the application protocol, and commands and responses between peers [how the management data is communicated between the agent and manager process]
  4. Functional [解决驻留在NMS的network management applications] FCAPS model: (Fault, Configuration, Accounting, Performance, Security)five areas of function

SNMP：IETF community； CMIP：OSI community

（Simple Network Management Protocol； Common Management Information Protocol）

Network management architecture组成：

  1. Management Station：network manager’s interface
  2. Management Agent：network-management software module，包含：
    • Polling (solicited) the management station requests data from the agent and the agent responds with the requested data
    • Trapping (unsolicited) a data gathering method that thresholds，If this threshold the managed device will send an alert message to the management station.

Management Information Base：表示方法为从上到下树节点标号，中间用”.”分隔

Severity level：0最严重，7最不严重

0 - Emergencies；1 - Alerts；2 - Critical；3 - Errors；4 - Warnings；5 - Notifications；6 - Informational (ISO syslog default level)；7 - Debugging

Lec 1

Spanning-tree protocol operations (STP)

Step 1: Elect a Root bridge

拥有最低BID的为Root bridge （Root bridge没有Root ports，只有Designated ports）

Step 2: Elect the Root ports

Non-root bridge选一个到Root bridge路径成本最低的端口，如果多个端口具有相同的最低路径成本，则接收到具有最低 Sender Bridge ID，若Sender Bridge ID仍一样，则选最低的Sender Port ID的端口成为 Root 端口

Step 3: Elect the Designated ports

每个网段上选择一个到根网桥的路径成本最低的指定端口，如果多个端口具有相同的最低路径成本，则BID最低的端口将成为指定端口。

同一Switch上，Root端口连向Root bridge，Designated端口连向树的子节点Switch

Step 4: Elect the Non-designated ports

STP 选择所有未使用的端口作为非指定端口。一个Designated 上级端口对应一个下级Non-designated port，所以一条逻辑上被截断的线只有一个port blocked by STP

How many ports are blocked by STP?

= 总端口数 - (Switch数 - 1) - 总网线数

Lec 2

VLAN communications

Intra - VLAN：同一VLAN内，第二层；Inter-VLAN：VLAN之间，第三层

连接两个VLAN：router(gateway) 或 layer 3 switch（通常用后者）

​ ⬆️若router连2个VLAN的物理port：必须trunk

​ ⬆️陷阱：只有switch的port成为trunk port；route是第三层，不可称为trunk port

Gateway redundancy：一个VLAN仅有一个gateway是active，其余均为它的stand by【不使用物理gateway IP和MAC，而使用virtual IP和MAC】

VLAN内通信计算题

首先按照VLAN划分为VLAN个数个逻辑拓扑，分别STP，再在同一VLAN拓扑内找路

VLAN间通信计算题

首先按照VLAN划分为VLAN个数个逻辑拓扑，分别STP，然后使用发送方active gateway

Lec 3

找Wireless Network数量

按照SSID判断，同SSID就是同一Wireless Network

WLAN topologies（3种）

Ad hoc mode：多个无线客户端无需AP即可连接，如PC连PC。分为IBSS 和 ad hoc network

Infrastructure mode：所有无线客户端通过一个或多个AP 互连。包含两种：

  • BSS：一个AP，BSSID=AP的MAC
  • ESS：多个AP，ESSID简称为SSID

Radio power

radio power单位：W 或 mW=10⁻³W

radio power比较用单位dB

dBw：以 W 作为Signal/Reference的单位；dBm：以 mW 作为Signal/Reference的单位

Power (in dB) = 10 * log₁₀ (Signal / Reference)

Lec 7

Committed Information Rate (CIR) 保证

Excess Information Rate (EIR) 非保证 单位Kbps

Committed Time (Tc)

Committed Burst (Bc) = CIR x Tc

Excess Burst (Be) = EIR x Tc 单位Kbits

▪ If the counter under Bc, the frame is not eligible for discard (DE=0).
▪ If it puts the counter over Bc, the frame is eligible for discard (DE=1), indicating that frames may be dropped if there is congestion or there is not enough capacity in the network.
▪ If the counter over Bc + Be, the frame is discarded.
▪ At the end of each Tc seconds the counter is reset. The counter may not be negative, so idle time cannot be saved up.

将N用户连接起来，使用Virtual circuit数量：

Min = N-1

Max = C²n = N(N-1)/2!

类型：

Full mesh：每个站点都连接到每个其他站点

Star topology (hub and spoke)：每个远程站点都有一个通过单个 VC 访问frame relay的链接。Hub有一个带有多个 VC的访问链路，每个远程站点一个

Partial mesh：比Star topology多，但少于Full mesh
