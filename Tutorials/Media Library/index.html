管理你的个人媒体库

发表于2022-03-24|更新于2022-04-20|教程

Q&A

我的媒体文件已经整整有条了，为什么还要整理呢？

不同播放器对文件命名的宽容度和关键词选取规则不一样，最终出来的识别效果也会不同。放到这个标题下也是一样，你的命名规则、文件夹结构未必能被元数据抓取器所识别，故而需要“整理”。

为什么要整理文件并组建个人媒体库？

为了使Emby、Kodi、Jellyfish等个人流媒体平台可以抓取、削刮并显示出来你的媒体的信息

我直接打开文件就可以播放了，为什么还要建立媒体库？

两个字，好看。可以参考另一篇文章中我搭建的Emby平台

你面对的不再是呆板的Windows文件资源管理器、文件列表、冗长的媒体文件名，而是海报墙和本地化的剧集信息预览。当你百无聊赖却又不知道重温什么片子的时候，你可以随手查看媒体库中某一部作品的资料然后确定你是否观看；更高级一点的，你甚至可以将资源库共享给你的朋友（需要将家庭内网接入公网，具体参考我的另一篇文章），告诉他们这是你的Netflix服务器。

文件的命名规则

电影的命名规则

在电影媒体库的文件夹中，将每部电影的媒体文件放入名为”电影名[空格](年份)”的文件夹中

电影媒体文件的文件名不是很重要，我个人建议[影片名][年份][清晰度][视频编码格式][音频编码格式][硬字幕字幕组来源(如果有)]，中间的空格和各项之间用英文句点替换

电视剧的命名规则

类似电影，每部剧放入一个文件夹中，不过每一季也要放入名为“S[两位数季号]“或“Season[空格][两位数季号]的文件夹中，每一集的文件名中要有”S[两位数季号]E[两位数集号]“。

具体季度、分集编号去TMDB查看

番剧的命名规则

番剧的命名规则基本和电视剧相同，主要问题是处理番剧中不属于正篇的分集、OVA、总集篇、剧场版：主要还是去TMDB查询，然后根据TMDB上的顺序，将这些放入与Season同级的Specials文件夹中

例如：

字幕的命名规则

emby支持的字幕格式：ass, srt, ssa, sub/idx, vtt

emby支持的字幕语言：https://www.loc.gov/standards/iso639-2/php/code_list.php

emby支持的字幕扩展：

- 强制字幕：forced/foreign
- 默认字幕：default

字幕文件需置于媒体文件的同级文件夹内，命名规则为[媒体文件名(不含文件扩展名)].[语言类型].[扩展类型(如果有)]
